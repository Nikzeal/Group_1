History

The C Programming Language was created in 1972 at Bell Labs by Dennis Ritchie, very tied with the Unix operating system, which was developed with his colleague Ken Thompson, both are used very widely even today. Originally it was implemented in assembly language for a PDP-7, incorporating suggestions from their colleagues, and eventually they ported it to a PDP-11.

Ken Thompson and Dennis Ritchie

Thompson wanted a programming language for developing to the new platform, creating a stripped version of BCPL ("Basic Compiled Programming Language"), which he made less wordy, which he called B. In 1971, Ritchie started to improve on B to utilize the features of the more powerful PDP-11 machine, which was in origin called NB ("New B"). Through to 1972 new additions were added, such as richer types, ability to generate pointers to other types, arrays of all types and types to be returned. It was renamed to C.

Before C

Before C came to light, B was developed as a typeless language for use on Multics, designed for recursive, non-numeric, machine-independent applications. They involved complex logical decision making, and processing of integers, characters, and bit strings. But with ASCII machines becoming more common, the typeless nature was seen as a disadvantage, which has brought forth the C programming language.

After C

Many later languages have borrowed from C, such as C++, C#, Java, JavaScript, Ruby and Rust. During the late 1970s, versions of C were implemented for a wide variety of mainframe computers, including the IBM PC, which has increased its popularity. In 1983, the American National Standards Institute (ANSI) formed a committee to establish a standard specification of C. In 1990, the ANSI C standard was adopted by the International Organization for Standardization (ISO).

The impact of C

C is widely used for systems programming in implementing operating systems and embedded systems applications for several reasons:

What uses C?

  • The code does not require many system features, and can be invoked with a boot code;
  • Statements and expressions map well on to sequences of instructions: low run-time demand on the system and faster execution times;
  • C can use many features of target CPUs, even of more esotheric types;
  • Direct control over memory allocation and deallocation;
  • Very mature and broad ecosystem: many drivers already exist in C, which reduces incentive to use another language.
